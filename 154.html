Getting Started with the DEFCON API

The absolute beginner's guide. Learn how to set up your environment, install the necessary SDK, and make your very first authenticated API call in minutes.

ðŸ•‘ 5 Minute Read Difficulty: Beginner

1. Introduction: Hello, DEFCON!

Welcome to the DEFCON developer ecosystem! This Quickstart guide is designed to get you from zero to your first successful API request as quickly as possible. We recommend using our official Python SDK for this tutorial, but the principles apply across all supported languages.

Pro Tip: All code examples on this page are copy-ready. Just click the copy icon when hovering over the code block!

2. Prerequisites

  • A **DEFCON Developer Account** (Sign up is free).
  • An **API Key** generated from your developer dashboard.
  • **Python 3.8+** installed on your system.
  • A functional **Terminal** or **Command Prompt** environment.

3. Project Setup

First, create a new directory for your project and navigate into it. We'll set up a virtual environment to manage dependencies cleanly.

Step 3.1: Create Project Directory

mkdir defcon-quickstart cd defcon-quickstart

Step 3.2: Initialize Virtual Environment

python -m venv venv source venv/bin/activate # On Windows, use: .\venv\Scripts\activate

*(This step is optional but highly recommended for dependency isolation.)*

4. Install the Official Python SDK

Use Pip, the Python package installer, to download and install the official DEFCON SDK.

Install Command

pip install defcon-sdk

Verify Installation

pip show defcon-sdk

The output should confirm the version and location of the installed package.

5. Your First Request: The Status Endpoint

Create a new file named main.py in your project directory. We will use the $\text{DEFCON API}$ to check the current system status.

Code: `main.py`

import os from defcon_sdk import Client # 1. Initialize the client using your API Key # For security, load the key from an environment variable API_KEY = os.environ.get("DEFCON_API_KEY") client = Client(api_key=API_KEY) # 2. Make the API call to the status endpoint try: response = client.status.get_system_status() print("API Call Successful:") print(response) except Exception as e: print(f"An error occurred: {e}")

Remember to set your $\text{DEFCON\_API\_KEY}$ environment variable before running.

Execution

python main.py

6. Expected Output & Verification

If your setup and request were successful, the output in your terminal should look similar to this JSON response:

{ "status": "operational", "message": "All systems nominal.", "timestamp": "2025-11-14T11:41:49Z", "service_version": "v1.2.0" }

**Congratulations!** You have successfully integrated with the $\text{DEFCON API}$. You are now ready to move on to more advanced tutorials like **"Authenticate Your First Request."**
